El Modelo de Caja CSS

  • Cada elemento HTML es una "caja" rectangular
  • Compuesto por: contenido + padding + borde + margen
  • El comportamiento por defecto puede causar problemas de diseño
  • El box-sizing determina cómo se calcula el ancho y alto
Problema común: "¿Por qué mi elemento de 200px ocupa 260px en la pantalla?"
/* Modelo de caja tradicional */
.element {
  width: 200px;
  padding: 20px;
  border: 10px solid;
  /* ¡Ancho total = 260px! */
}

Content-box (Comportamiento por Defecto)

  • Valor por defecto de box-sizing
  • El ancho/alto se aplica SOLO al contenido
  • Padding y borde se SUMAN al tamaño total
  • Fórmula: ancho total = width + padding-left + padding-right + border-left + border-right
¡Cuidado! Con content-box, el tamaño real del elemento es impredecible cuando agregas padding y bordes.

Border-box (Solución Moderna)

  • El ancho/alto incluye TODO: contenido + padding + borde
  • Padding y borde se RESTAN del espacio del contenido
  • Fórmula: ancho total = width (fijo)
  • Mantiene el tamaño total predecible
¡Recomendado! border-box hace que el diseño sea más intuitivo y predecible, especialmente en layouts responsivos.

Por qué border-box es mejor para Responsive Design

  • En diseños responsivos, necesitas elementos que ocupen el 100% del contenedor
  • Con content-box: width: 100% + padding = ¡overflow!
  • Con border-box: width: 100% se mantiene consistente
  • Facilita grids, layouts de columnas y componentes reutilizables

Cómo Aplicar box-sizing Globalmente

  • Nunca apliques border-box manualmente en cada elemento
  • Usa un selector universal en tu CSS base
  • Incluye pseudo-elementos (::before, ::after)
  • Compatibilidad: soportado en todos los navegadores modernos
/* Reset moderno para box-sizing */
*, *::before, *::after {
  box-sizing: border-box;
}

/* Opcional: heredar en componentes específicos */
html {
  box-sizing: border-box;
}
*, *::before, *::after {
  box-sizing: inherit;
}

Compatibilidad y Buenas Prácticas

  • Compatibilidad: IE8+, Chrome 4+, Firefox 29+, Safari 5.1+
  • Siempre incluye el reset global al inicio de tu CSS
  • No confundir con width: 100% + padding en imágenes
  • border-box no afecta al margin (espacio externo)
